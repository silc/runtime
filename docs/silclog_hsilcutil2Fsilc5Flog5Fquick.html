silc_log_quick

NAME

    void silc_log_quick(SilcBool enable);

DESCRIPTION

SilcLog makes use of libc stream buffering output, which means that it saves HD activity by buffering the logging messages and writing them all together every some minutes (default is 5 minutes).

Setting `enable' to TRUE will force SilcLog to write messages to the filesystem as soon as they are received. This increases the CPU activity notably on bigger servers, but reduces memory usage.

If you want to change the logging style on-the-fly, make sure to call silc_log_flush_all() after setting `enable' to TRUE.

Default is FALSE.
